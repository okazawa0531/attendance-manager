# 概要

勤怠を管理するためのアプリケーションです。

# 構成

| 構成要素 | 役割 |
|---------|------|
| フロントエンド | UIを担当します。管理者用ページと利用者用ページの2つを提供します。 |
| バックエンド | DBの操作を担当します。 |

# 機能仕様

## 管理者用機能

### ユーザー管理

- ユーザー一覧表示（氏名・メールアドレス・役割・登録日）
- ユーザー追加（氏名・メールアドレス・役割・初期パスワードを入力）
- ユーザー削除

## 利用者用機能

### 今日の勤怠

- 出勤打刻・退勤打刻（ボタン押下で現在時刻を自動入力）
- 出勤時間・退勤時間の手動入力
- 休憩時間（分単位）の入力
- ステータス選択（出勤 / 欠勤 / 休日 / 有給休暇）
- 備考入力
- 勤務時間のリアルタイム計算・表示
- 登録済みの場合は上書き更新

### 勤怠履歴

- 年・月セレクタによる月別切り替え（過去5年分）
- 表示月のすべての日付を一覧表示（土日は色分け）
- 未登録日はステータスを「未登録」として表示
- 月次サマリーの表示（出勤日数・有給休暇日数・欠勤日数・総勤務時間）
- 登録済みレコードの編集（モーダル）
- 未登録日への過去日付の新規登録（モーダル）

## ステータス種別

| ステータス | 説明 |
|-----------|------|
| 出勤 | 通常出勤日 |
| 欠勤 | 無断・届け出欠勤日 |
| 休日 | 土日祝などの休日 |
| 有給休暇 | 有給取得日 |
| 未登録 | 勤怠記録のない日（表示専用） |

# 環境

AWS 環境で動作させます。CDK による構築環境も作成してください。

## 要件

- 可能な限りコストを抑えてください。使用しなければ料金が発生しないことがベストです。発生しない構成が難しい場合は、発生するコストの目安を明示してください。
- DB は DynamoDB を使用してください。キャパシティはオンデマンドで設定してください。

# デザイン

目に優しい緑基調のデザインにしてください。
